# docs/fetch_pipeline.md — Data pipeline gotchas (fetch_data.py)

> Loaded on demand from CLAUDE.md. Read when editing `fetch_data.py` (sources, off-FP seeding, probable starters, matchup dates, lineup efficiency). Rules here are as authoritative as CLAUDE.md.

**`get_lineup_efficiency(league, my_team_name, mode)`** (`fetch_data.py`) — reconstructs MY team's **daily** lineup to quantify start/sit opportunity cost (mechanism + rationale in NOTES). **Both modes span the FULL matchup period, not just the current calendar week.** `mode="prev"` audits the last completed matchup (recap post-mortem); `mode="current"` audits the in-progress matchup **first Monday → yesterday** (today excluded as incomplete). Returns `{}` on a Monday with no completed days. The span comes from ESPN's schedule — a matchup entry's **`home.pointsByScoringPeriod` keys ARE the exact days that count** (robust for the 14-day All-Star matchup; falls back to this-Monday±7 on fetch failure). Computed in `main()`, stored as `lineup_efficiency` (prev) + `lineup_efficiency_current` (current). Per-day slot comes from `mRoster?scoringPeriodId=<day>`. **Four findings:** (a) **batter bench leakage** — R/HR/RBI/SB in a `BE` slot, reported **NET of the weakest startable bat you'd have benched** (feasibility of a no-bench slot-in is a max-bipartite-matching check `_full_match` on `eligibleSlots` — see NOTES); net uses only *notable* days (HR/SB/2+R/2+RBI). (b) **pitcher blowups** — an active-slot start with 5+ ER (or 4+ ER in <3 IP) that counted, cross-refed against `recent_activity(size=150)` drops → "imploded then dropped". (c) **idle active hitters ("wasting space")** — 0-AB active-slot hitter, **counting only games his MLB team actually played** (schedule-gated via `_ESPN_PROID_TO_MLBID`; fetch-fail → count all); fires only on **3+ consecutive idle games** OR an AB in **< 50%** of active games (min 4). (d) **benched-SP leakage** — a **good start** (a QS *or* 6+ K *or* a W; start detected via daily GS stat id 33, or 5+ IP as a proxy) put up while sitting in a `BE` slot, so it never counted. **Only the counting cats K/QS/W are netted** — NET of the weakest **same-day active pitcher** you'd have benched to start him (same `_full_match` feasibility on the pitcher slots `PIT_IDS`={13,14,15}; a rest-day active arm contributes 0 → full gain, so the common "your active SP was resting" case reads as an open slot). **Ratios (ERA/WHIP) are NEVER netted** — shown as directional prose in the line only (an active-blowup-style honesty choice; netting whole-staff IP-weighted ratios is fragile). The headline `net_pit` floors each cat at 0 and sums only surfaced (net-positive) starts. Returns `bench_sp` (list) + `net_pit` (`{K,QS,W}`) alongside `bench`/`net`. **OPS is never netted** (rate stat — benching an 0-4 day helps OPS). Pitcher/hitter split via `eligibleSlots`. Broad try/except → `{}` on any failure. Standalone opponent + head-to-head version: **`bench_leakage.py`** (batter/blowup/idle only — no benched-SP finding there yet).

**`fetch_todays_games(league)`** (`fetch_data.py`) → snapshot field `todays_games` — today's real MLB games enriched with which **rostered** players (any fantasy team) are in each. One `statsapi /schedule?date=today&hydrate=probablePitcher,broadcasts(all)` call. **Roster→game join is on MLB team id** (robust vs abbrev quirks): espn_api exposes a player's club as a `proTeam` **abbrev string** (`'Sea'`, NOT a numeric `proTeamId`), mapped abbrev → ESPN id via **`espn_api.baseball.constant.PRO_TEAM_MAP`** (inverted) → MLB id via `_ESPN_PROID_TO_MLBID`; players bucketed by MLB id, matched to each game's `teams.home/away.team.id`. Each involved player carries `is_p` (pitcher?), `is_sp` (the game's **confirmed probable starter**?), and `is_rp` (**reliever by USAGE ROLE**, not position — via `_espn_is_reliever(pl)`: ESPN season GS/GP ≤ 0.20 with ≥ 5 apps → RP, ≥ 0.80 → SP, ambiguous/thin → `eligibleSlots` RP-only). `is_rp` lets the renderer COUNT relievers (a save/hold chance moves the matchup) while skipping a starter on his off-day. Per game: `gamePk`, `game_time_utc`, `home_name`/`away_name` (full names — renderer maps via `_FULLNAME_TO_ABBREV`), `home_prob`/`away_prob` (`""` when TBD), broadcasts split into `national_tv` (list, `isNational`), `home_tv`/`away_tv` (each side's local RSN), and `involved` (`[{name, FantasyTeam, is_p, is_rp, is_sp}]`). Games with zero rostered involvement are dropped. Broad try/except → `[]`. Consumed by `send_digest.build_todays_games_section` + the Briefing "📺 Tune in" teaser.

### Data sources
- **Pending trades aren't in `espn_api`.** `get_pending_trades(league, my_team)` hits the raw endpoint `.../leagues/{id}?view=mPendingTransactions` with the same `espn_s2`/`SWID` cookies. **Trades live under the top-level `pendingTransactions` key, NOT `transactions`** (that key is empty in this view). Each `TRADE_PROPOSAL` item has `fromTeamId`/`toTeamId` (direction) + `type=="TRADE"`; `memberId == my SWID` ⇒ I proposed it (`incoming=False`). Broad try/except → `[]`.
- FanGraphs returns 403 — never use directly. pybaseball works (handles headers).
- `pitching_stats()` (FanGraphs leaderboard) → 403. Use `pitching_stats_range()` (Baseball Reference) — but it has no `HLD` column.
- SVHD comes from ESPN via `get_pitcher_espn_svhd()`, reading `pl.stats[0]['breakdown']`, which uses **string keys** (`'SV'`, `'HLD'`, `'SVHD'`, `'K'`, `'W'`, `'OUTS'`, `'ERA'`, `'WHIP'`, `'GP'`, `'GS'`) — not numeric IDs. Called at fetch time for all rostered + FA pitchers.
- ESPN season stats (`ESPN_SV`/`ESPN_K`/`ESPN_W`/`ESPN_IP`/`ESPN_GS`/`ESPN_GP`/`ESPN_SVHD`) are stored on **all** dataset rows so send_digest can use season counts for players who only appear in short-range FantasyPros datasets. `ESPN_SVHD`/`ESPN_SV`/`ESPN_HLD` override `SVHD`/`SV`/`HLD` on `Dataset==YEAR` rows; `ESPN_HLD` is then dropped but `ESPN_SV` stays on all rows (the only way `save_role_watch` distinguishes a real closer from a holds-only reliever outside the FP top-300). Use `_n(r.get("ESPN_SVHD")) or _n(r.get("SVHD"))` (not `if >= 0`) — `_n` floors negatives to 0 so `>= 0` is always true.
- xFIP / CSW are unavailable from FantasyPros (and FanGraphs — 403). Baseball Savant (via pybaseball) supplies predictive pitcher stats, merged on `PlayerName`: `xERA` + `xwOBA_against` (`get_savant_pitcher_expected`, absolute values), `WhiffPctile` (a league whiff **percentile** 0–100, not a rate, from `get_savant_pitcher_skill`), and `BarrelPctAllowed`/`HardHitPctAllowed`/`AvgEVAllowed` (`get_savant_pitcher_contact`). Coverage ≈ 90–99% of the FP top-300; missing rows fall back to raw ERA/K% cleanly.
- **`WhiffPct` (raw overall whiff %, DISPLAY-ONLY)** — via `get_savant_pitcher_whiff` (pybaseball `statcast_pitcher_arsenal_stats`), pitch-type rows aggregated by pitches-weighting per `player_id`. A raw 0–100 **rate**, DISTINCT from `WhiffPctile` (a 0–100 **percentile**). Merged on `PlayerName` via `merge_on_name`. **Never feed it into `pitcher_score`/`rp_score`** — `WhiffPctile` already drives the K component, so raw whiff% would double-count. Displayed only: a muted `whiff NN%` subline under the K% cell in the compacted My Upcoming Starts + FA SP tables (`_whiff_sub`), and a **Whiff%** column (green ≥ 30) in Pitcher Hot/Cold (7 cols; `score_reveal` colspan 7).
- ESPN injury statuses are `TEN_DAY_DL`/`FIFTEEN_DAY_DL`/`SIXTY_DAY_DL` — not `IL`/`OUT`. Constant `_DL_STATUSES` covers all. FA views and positional breakdown exclude all DL-status players. `ESPN_Status` (rostered — `pl.injuryStatus or "ACTIVE"`) is merged for **both** hitters and pitchers (`build_hitter_data`/`build_pitcher_data`); `send_digest._get_injury_status` reads it first, falling back to `FreeAgentInjuryStatus` (FA) then a FantasyPros-embedded-tag parse.
- **FA pull size (`_FA_PULL_SIZE = 1000` in fetch_data.py, all 3 `league.free_agents()` calls: `get_pitcher_fa`/`get_hitter_fa`/`get_pitcher_espn_svhd`).** ESPN's default `size=50` only returns the top-50-by-%-owned free agents — a live check found this covers just ~13% of the realistic FA candidate pool (bounded by FantasyPros' own top-300-per-role scrape, ~285 unrostered names), so a moderately-owned injured player (5–10% owned) routinely falls outside it. When that happens, `FreeAgentInjuryStatus` merges to a default blank — indistinguishable from a genuinely healthy, checked free agent — so `_DL_STATUSES` filters can't catch him and he shows as a clean pickup (confirmed live: Blaze Alexander, IL10, rank 86 by %-owned, invisible at `size=50`). `size=1000` gets ~100% coverage at ~1.6s/call (negligible in the ~60s pipeline). **Don't lower this** without re-running the coverage check. `FA_Matched` (bool, both roles — `merge_on_name(...).notna()` on `FA_Position` before it's dropped) records whether a given FA row was actually returned by the pull, so `data_coverage.py`'s `fa_status` group can catch a future regression of this gap.

### Weekly matchup is Monday–Sunday (or Monday–Sunday×2 for All-Star break)
`week_end_str` comes from `matchup_end_date` in the snapshot (not a hardcoded `today + 6 - weekday`). `fetch_data.get_matchup_dates(league)` derives the window in two parts. **START** = the earliest DAILY scoring-period id actually scored in the current matchup, from `pointsByScoringPeriod` (`_current_matchup_start_sp`, raw `mMatchupScore` view — the same authoritative day-span source `get_lineup_efficiency` trusts), converted to a date via `today - (today_sp - start_sp)` (daily SP ids are consecutive days). **Do NOT anchor START at "this Monday" (`today - weekday`)** — that lands a full week late for the **2nd week of a multi-week (All-Star) matchup**, zeroing `days_elapsed`/`game_days_elapsed` so every counting-cat projection stacks a full period onto already-banked stats (see NOTES). Falls back to the Monday anchor only when `pointsByScoringPeriod` is unavailable. **LENGTH** uses the `finalScoringPeriod` surplus arithmetic (`remaining_daily = final_sp - true_start_sp + 1`; `period_days = 7 + min(max(0, remaining_daily - remaining_regular*7), 7)`) — NOT `len(matchup_periods[id]) * 7`, because ESPN's `matchup_periods` dict is **degenerate in this league** (period 15 → `[15]`, all length-1, so it never reveals the 14-day span). Anchoring at the true start is what makes the surplus resolve to 14 mid-break. Stored in snapshot as `matchup_start_date`, `matchup_end_date`, `matchup_period_days`, `next_matchup_end_date`, plus **`matchup_game_days`/`matchup_game_days_elapsed`** (real MLB game days in the window via a `statsapi /schedule?gameType=R` query, excluding the All-Star break; drive `elapsed_frac` in Category Pulse/`classify_categories`) plus **`team_hit_sched_frac`** (`{team_name: fraction}`, built from the SAME schedule response — each fantasy team's roster-weighted fraction of window bat-games still to come, via a `proTeam→ESPN→MLB id` join over `league.teams` hitters; feeds `send_digest.compute_hit_proj` so hitter counting-cat projections respect real off-days/doubleheaders instead of a league-wide time fraction; `{}` on fetch failure → graceful fallback). Dates past `week_end_str` get a `NEXT WK` badge. **Bullet 2 scopes both `confirmed` and `thin_days` to `PSP_Date <= week_end`** so its count matches the KPI.

### Probable starters — sourced from ESPN's public MLB scoreboard (full week)
`get_probable_starters(days=SP_DAYS_OUT)` pulls purely from ESPN's public scoreboard endpoint (`_ESPN_SCOREBOARD`, one call per day over the window). **Rationale (load-bearing):** the MLB Stats API only *confirms* probables ~48h out, so the prior "MLB confirmed + homemade rotation-walk projection" approach left the mid-week days (Thu/Fri) a **dead zone** — too far for MLB confirmation, too near for the walk's `_PROJ_MIN_DAYS` gate — so FA SP / two-start detection came up empty there. **ESPN projects a probable for EVERY game a full week out** (its own rotation model), which fills the whole window (verified: Fri jumped 3→29 arms). Each game's home/away `probables[0].athlete.displayName` → an accent-stripped `PlayerName`; ESPN team `displayName` matches MLB's `team.name` **exactly for all 30 clubs** (verified), so `PSP_HomeVAway` keeps the `vs/@  <full team name>` form the opponent-OPS merge (`get_opponent_ops` on `team.name`) and `opp_logo` already expect. **ESPN exposes NO confirmed/projected flag**, so `PSP_Projected` is inferred from days-out: `< _PROJECTED_MIN_DAYS_OUT` (2) → confirmed (today+tomorrow, which MLB has firmed up), else projected. The daily re-fetch supersedes a projection with the real line as the date approaches; confirmed (`False`) sorts before projected (`True`) so `_attach_start_lists`'s dedup keeps a confirmed entry over a projected one for the same pitcher/date. A pitcher appearing on ≥2 dates in the window is a two-start week — `_attach_start_lists` folds all dates into `PSP_Dates`; scalar `PSP_Date` stays the earliest. Broad try/except per day → empty DataFrame on total failure (downstream degrades to "no upcoming starts", same as an MLB outage before). **The old MLB two-call + rotation-walk path (`_get_team_recent_starts`, `_probable_starters_live_feed`, `_ROT_RECENCY`/`_PROJ_MIN_DAYS`) was deleted** — do not reintroduce a rotation walk; ESPN already projects the week.
