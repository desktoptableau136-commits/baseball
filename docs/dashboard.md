# docs/dashboard.md — Dashboard (dashboard.py)

> Loaded on demand from CLAUDE.md. Read when editing `dashboard.py`. Rules here are as authoritative as CLAUDE.md.

**`dashboard.py`** reads the same snapshot and builds a **single-viewport, zero-scroll "command dashboard"** — `previews/dashboard_{team_slug}.html`, sized for a **1440×900 laptop with NO scrolling**, glance-able coverage of every digest topic. **Imports `send_digest` and reuses its scoring/projection functions verbatim** (every number matches the digest). `build_context(snap, my_team)` derives its values via the SHARED send_digest helpers (`sd.prepare_scoring` — the canonical order-sensitive calibration sequence + percentile pools + scarcity — then `sd.build_recent_indexes`/`sd.claimed_today`/`sd.parse_matchup_window`/`sd.compute_pit_proj`/`sd.compute_week_finishes`/`sd.roster_hot_cold_counts`, the same ones `build_email` calls), then rebuilds FA lists, `weekly_avgs/std`, `pit_proj`, `classify_categories`, `positional_breakdown`, `opponent_week_intel`, `save_role_watch`, `_roster_suggestion`, `make_sparkline`, etc. Also computes `trades` (`sd.find_trades_combined`).
- **Desktop layout:** top KPI bar (8 chips) + fixed 3-col CSS grid (`#wrap{position:fixed;inset:0;display:flex}` — **load-bearing, see NOTES**; `#grid{display:grid;grid-template-columns:1fr 1fr 1fr;grid-template-rows:minmax(0,1fr)}`, `.col{height:100%}`) + a bottom **legend strip** (`render_legend`/`#legend`). Tiles `overflow:hidden` and clip to the top-N rows that fit (never scroll) — **the ONE exception is the Lineup Watch pane inside the Weakest Spots tile** (its own `overflow-y:auto`). Per-tile `flex` weights give content-heavy tiles more height — don't equalize blindly. **Verify with a headless screenshot at 1440×768 (not just 900)** — browser chrome shaves usable height (see NOTES).
- **Legend strip (`render_legend` desktop bar / `render_legend_panel` tablet tile):** shared key items come from **`_legend_items()`** (single source, so the two renderings can't drift). Desktop = a slim one-line `#legend` key pinned to the foot of `#wrap` (a `flex:0 0 auto` sibling AFTER both `#grid` and `#gridt`). Tablet/phone: `#legend` bar hidden (`#legend{display:none}` ≤1100px) and the same key renders as a normal `_tile("Key", …)` placed last in the right tablet column `colt_r`. Defines every badge/marker: score pill, ▲▼◆ proj W/L/T, ⚡ toss-up, win %, ×2 two-start, QS, 5K+, ⚠, `$`/`▼`, PWR, SB, 🔥/❄. `_legend_chip(glyph, color)` reproduces the tinted `_hit_badge` look; keep it in sync with the real badge colors (QS cyan, two-start blue) when they change.
- **Column assignment:**
  - **col 1** = Today's Games (`render_tv_games`, TOP; favorite team ★-pinned first + next highest-overlap game, 2 max; reuses `sd._rank_todays_games(pin_favorite=True)` + `sd._fmt_game_time_et` + the same role-aware badges as the digest; `flex=1.05`, `''` when no games) + Category Pulse (`flex=1.6`; 4×3 compact 12-cat grid; my/opp values + `proj X / Y` sub-line + win%/⚡ + ▲▼◆ via `_project`/`_cat_win_prob`) + Weakest Spots/Lineup Watch (`flex=1.0`; fixed Weakest-Spots holes on top + a scrollable `overflow-y:auto` Lineup Watch pane below — `build_bench_watch`, empty on `--team`). **`render_holes` shows only GENUINE needs — bottom-third rank** (`_is_need`: `rank >= n − third + 1`, `third = round(n/3)`), the SAME need gate as `_roster_suggestion`/Trade Radar, then `[:3]`. Fewer than 3 needs → fewer rows; zero needs → "Balanced roster." fallback. Green upgrade is judged vs `my_avg` (`fsc − my_avg`).
  - **col 2** = My Pitching · Hitting Hot/Cold · **Trade Radar / Trades** (`render_trade_radar` — top **2** swaps, DISTINCT partners preferred, each a **3-column mini-card**: col 1 = partner + `you win`/`even`/`you pay up` value tilt + a `realistic`/`aggressive ask` acceptance sub-line (`sd._trade_tilt`, rival's-POV) + sell-high/buy-low thesis; col 2 = Give; col 3 = Get. Players one-per-line with a `_mini_badge` score pill (hover `title` = `_score_tip`, NOT tappable) + canonical `$`/`▼` + `(C)` position chips. **A REAL incoming trade offer LEADS this tile**: `build_context` runs `sd._grade_pending_trades(...)` → `pending_incoming`; when non-empty the tile retitles to **"Trades"** and renders each offer as a top mini-card (📥 OFFER TO REVIEW · partner · `sd._verdict_pill` · red `expiry_str` · Give/Get · 💡 `Counter:` line on a COUNTER), then **ONE speculative radar idea below** (`max_radar = 1`) from a team not in `offer_partners` (filtered out of the radar loop; cap checked BEFORE appending). When an offer leads, the body scrolls (`overflow-y:auto`). Empty on `--team`/no offer → plain "Trade Radar" with 2 ideas).
  - **col 3** = Recommended Moves + Save-Role · FA Radar (top **2** per role) · Season (standings/roto/`make_sparkline`/weekly-finishes strip). **Recommended Moves** (`render_moves`) shows a `_mini_badge` score pill after every add AND drop name — `build_context` passes `sd._roster_suggestion(..., pill_fn=_mini_badge)`; the `pill_fn` callback is off by default so the digest bullets stay pill-free.
- **My Pitching badges:** each upcoming start shows projected `IP·ER·K`, a blue `×2` two-start marker BEFORE the matchup date, and cyan **QS** / yellow **5K+** / ⚠ / **`$`/`▼`** (`_reg_chip8`, 8px, off `sd.pitcher_regression_flag`) badges AFTER it — QS/5K+ driven off the SAME `sd._proj_line_vals(r)` as the displayed line. FA Radar Starters + Relievers render `sd.pitcher_regression_badge` (Starters also `sd.blowup_badge`); Weakest Spots renders it on pitcher-position rows (else `sd.hitter_badges`).
- **Responsive (below 1100px):** media queries relax the fixed pane WITHOUT touching the desktop layout — the pinning inline styles are overridden via **classes + `!important`** (`.tile`, `.tile-body`, `.pulse-grid`, `.topbar`, `.topbar-chips`, `.chip`; inline otherwise wins). **Tablet/phone use a SEPARATE container, not a reflow** (why two grids, not `order`/multicol — see NOTES). `build_dashboard` renders each tile ONCE into two containers: `#grid` (desktop 3-col) + `#gridt` (tablet: two `.colt` flex columns), one `display:none` per breakpoint. Tablet `.colt` split (height-balanced): left = Today's Games · Pulse · Moves · FA; right = Pitching · Hitting · Weakest · Trade Radar · Season · Key. ≤1100px: `#wrap`→`position:static`+`overflow-y:auto`, `#grid{display:none}`, `#gridt{display:flex}`, tiles content-sized, pulse 3-wide. ≤700px (phone): `#gridt{flex-direction:column}`, larger font, pulse 2-wide.
- **When adding a tile:** render it once in `build_dashboard`, place it in BOTH `#grid` (a `.col`) and `#gridt` (a `.colt`), and re-check the two `.colt` heights stay balanced.
- **MLB team logos** (`sd.team_logo(r.get("Team"), size)`, `""` on unknown abbrev) precede the player name in every player-listing tile: My Pitching (14px), Hitting Hot/Cold (14px), Weakest Spots (13px), FA Radar (13px), Trade Radar (12px, give+get), Today's Games (13px, both teams). Recommended Moves + Save-Role skipped (prose/plain-name).
- **`--email`** sends the dashboard as a STANDALONE ATTACHMENT via `send_dashboard_email` (reuses send_digest Gmail creds); inline body is a pointer only (Gmail strips the `<style>` holding the layout). **Tab title** `Dashboard — {my_team}` (digest mirrors as `Daily Digest — {my_team}`). No glossary. **The daily dashboard ships as an ATTACHMENT ON THE DIGEST EMAIL** (`send_digest.py --with-dashboard`, one fetch/one email — see Automation); `daily-dashboard.yml`'s schedule is retired (`workflow_dispatch`-only). Still runnable locally standalone (`python dashboard.py --email`).
