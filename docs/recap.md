# docs/recap.md — Weekly recap (weekly_recap.py)

> Loaded on demand from CLAUDE.md. Read when editing `weekly_recap.py`. Rules here are as authoritative as CLAUDE.md.

**`weekly_recap.py`** reads the same snapshot on Mondays, builds a full-league recap HTML email: Week N Highlights · My Matchup (full 12-cat table) · Lineup Efficiency (`build_lineup_efficiency`) · League Scoreboard (all 6 matchups) · Weekly Roto Rankings · Top Performers (rostered + hot FAs, hitters/pitchers side-by-side via `_two_col`) · Standings & Luck · Season Trajectory · **Season Roto Rankings** (`build_season_roto_rankings` — heat-mapped 12-cat grid aggregated over EVERY matchup: teams **ranked** by cumulative `Roto_Score`, **displayed** value = the true season-to-date stat from ESPN's cumulative `mTeam` view (`season_cat_totals`), NOT a sum/average of weekly values. Ranking and value are independent by design — for rate cats {OPS,ERA,WHIP} the season stat can't be recovered by averaging weekly values; falls back to the summed/averaged weekly value only when a season total is missing). **Imports `send_digest`** for all shared code (colors/styles, `_n`/`_fmt_refresh_time`/logos/`band_divider`/`section_head`/`luck_standings`/`send_email`, and the season Trajectory + Roto Rankings builders — the old hand-copied versions had already drifted and are deleted); only recap-specific rendering lives here. Output: `previews/recap_week_N.html`. GitHub Actions: `.github/workflows/weekly-recap.yml` (Monday 15:30 UTC).

**`build_commissioner_story`** (`weekly_recap.py`) — Week N Highlights at the top of the recap. Two-column: commissioner prose (left, 60%) + stat sidebar (right, 40%). Finds (a) weekly roto winner, (b) hitter of week (rostered, best OPS, min 10 AB from `prev_week_hitting`), (c) pitcher of week (rostered, best ERA, min 8 IP from `prev_week_pitching`), (d) best available FA hitter. `prev_week_hitting`/`prev_week_pitching` are snapshot fields over the exact previous matchup Mon–Sun window — distinct from rolling `recent_hitting`(7d)/`recent_pitching`(15d) (fallback only). **`build_top_performers` also runs on `prev_week_*`** (subtitle shows the actual week dates; pitcher table shows **K** not G). **Both run names through `_fix_mojibake` before the season-roster lookup** — `prev_week_*` names arrive mojibaked, so without the fix the `_name_key` fallback misses the roster row and the player wrongly shows as a free agent. `_fix_mojibake` cleans TWO literal-backslash artifacts: `\xNN` escapes (`Eury P\xc3\xa9rez` → `Eury Pérez`) AND backslash-escaped apostrophes (`Ryan O\'Hearn` → `Ryan O'Hearn`, which carries no `\x` so it's cleaned before the `\x` early-out). The apostrophe cleanup is load-bearing since the shared canonical `_name_key` PRESERVES apostrophes — an uncleaned `O\'Hearn` would miss the roster row. Prose uses named historical benchmarks (Bonds 1.422 OPS, Gibson 1.12 ERA, deGrom 1.70 ERA, Ryan 383 K, etc.). Sidebar cards: fantasy logo + MLB team logo (`_mlb_logo`, ESPN CDN) + slash line / K / QS / categories led. Roto weekly win count = weekly top-roto-score count, NOT H2H W-L. MLB abbrev from season rows' `Team` field; `_MLB_ABBREV_ESPN` maps Baseball Reference outliers (TBR→tb, KCR→kc, SFG→sf, WSN→wsh, etc.).

**`build_league_scoreboard`** (`weekly_recap.py`) — all 6 matchups, one card each, 12-cat grid. Each team's identity (logo + name + W–L–T) lives in its **row label** (first column), NOT a top banner. Every card is a **separate `<table>`** using `table-layout:fixed` + a `<colgroup>` (150px label col + 12 equal cat cols) so the label→category axis lines up card-to-card. Long names ellipsis-clip. Category headers are **neutral** (`MUTED`), not green/red (even-handed full-league view); the **outlined value** (`outline:1px solid`) marks each category's winner. `_row` renders both teams (top = `my_team`/`my_val`, bottom = `opp_team`/`opp_val`).

**`build_lineup_efficiency`** (`weekly_recap.py`) renders `lineup_efficiency` (prev matchup) as a **LINEUP EFFICIENCY** band after My Matchup: red "LEFT ON THE BENCH" net headline, per-player bench cards (each notable day + swap target `[vs Name H-AB]`/`[open slot]`), a **"STARTS LEFT ON THE BENCH"** K/QS/W headline + per-pitcher benched-start cards (from `bench_sp`, each start line + `[open slot]`/`[vs Name IP]`), active-slot blowups with the drop flag, then an **"IDLE — WASTED ACTIVE SLOTS"** group. **`build_bench_watch`** (`send_digest.py`) renders `lineup_efficiency_current` (matchup-to-date) as the compact **Lineup Watch** callout (section 1b, after `alert_section`) — net-recoverable batter line + a **benched-SP line** (`+N K · +1 QS · +1 W from starts left on your bench`, ALL net-positive benched pitchers — no cap; the digest/dashboard row list mirrors the recap's uncapped list, aggregated as "K · QS · W across N starts" when multi-start) + implode-then-drop pitchers + a "Wasting active space" idle-hitter line (top 3); silent on a clean matchup; header "matchup to date". Gated on `not override_team` (snapshot stores only MY daily lineup).
